CodeProbe HTML — heuristic code analysis in the browser

Browser interface for local analysis of Python, JavaScript, Bash, C, C++, C# and Markdown, with JSON and text export, local history, lightweight syntax highlighting and configurable metric weights and thresholds. The analysis is probabilistic and should be read alongside oral examination, version history and assignment context.

Transparent package • browser UI Engine: Python in the browser Engine initialising

Course use policy

This score is a formative signal, not proof of misconduct. Student-authored code should normally remain below 60%. Scores above 60% should trigger revision, re-analysis and, where required by the module, a brief disclosure describing any AI assistance used during development.

Target: below 60% Use with Git history and oral explanation
Use on authored code only Check only the files written by the student for the assessed task.
Exclude non-student material Leave out starter code, third-party libraries, generated files, minified assets and documentation.
Above 60% means revise Rework naming, structure, comments, repetition and flow, then run the analysis again.
No automatic penalty Where concern remains, read the report alongside commits, design notes and a short code walkthrough.
Ctrl/Cmd+O opens a file · Ctrl/Cmd+Enter runs analysis · Ctrl/Cmd+S exports JSON
